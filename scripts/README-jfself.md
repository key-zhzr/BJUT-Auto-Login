# 计费系统离线采集

这组脚本用于在只能连接校园网、无法同时使用 Codex 的环境中采集
`jfself.bjut.edu.cn` 的登录协议和完整模块结构。它只登录一次，并以只读
GET 请求采集控制台、四类账单查询、报停/复通、复通记录、预约套餐、消费
保护、设备、资费、充值入口、修改密码和密码保护页面。它还会读取已人工确认
为查询用途的账单分页、业务记录、设备和资费列表接口；不会执行充值、套餐
变更、设备下线、密码修改等写操作。采集结束后会注销临时计费会话。

## 使用

在仓库根目录打开终端：

```bash
chmod +x scripts/capture-jfself.sh
./scripts/capture-jfself.sh
```

按提示输入计费账号和密码。密码输入不会显示，也不会写入参数、文件或日志。

脚本默认先使用系统 DNS；失败后再将 `jfself.bjut.edu.cn:443` 固定到
`172.21.0.16`。也可以手动选择：

```bash
./scripts/capture-jfself.sh --route system
./scripts/capture-jfself.sh --route fixed
```

仅当输出明确提示 TLS 证书校验失败，且已经确认连接的是可信校园网络时，才使用：

```bash
./scripts/capture-jfself.sh --route fixed --insecure
```

## 输出

输出均位于 Git 忽略的 `billing-capture.local/`：

- `*.private/`：原始页面，可能包含账务信息，不要发送、不要提交；
- `*.share/`：自动脱敏后的目录，可先人工检查；
- `*.share.zip`（或 `.tar.gz`）：确认内容后发送给 Codex 分析。

脚本会删除临时 Cookie。脱敏包会去除已输入账号、密码、Cookie、`checkcode`、
会话/CSRF 令牌、姓名、设备地址、手机号、身份证号和邮箱；仍建议在发送前打开 `report.json`
和 `sanitized/` 人工确认一次。若登录失败或出现人机验证码，脚本不会自动
重试，以免触发账号锁定。

脚本会先校验页面返回的登录表单，并按浏览器顺序加载同域静态资源、验证码图片与
品牌信息，再向登录表单中经过同源校验的 `/Self/login/verify`（可带
`jsessionid` 路径参数）提交一次登录。登录 POST 不自动跟随重定向：脚本会
拒绝可能重放账号密码的 307/308，只将经过 HTTPS 同源校验的 301/302/303
目标改用 GET 获取。
若登录页已经显示人机验证码，脚本会在提交账号密码前停止，避免继续累计失败次数。
服务端的隐藏 `checkcode` 通常为四位数字，但可能省略前导零而返回一至三位；脚本会
原样提交页面值，不自行补零。

脚本只读取模块 HTML、同源 JS/CSS，以及已经从页面代码中确认语义的 GET
白名单：控制台历史记录、在线设备、下线提示、无感知状态，四类账单分页，
报停/复通/套餐办理记录、设备列表和资费列表。日期查询窗口限定为最近 60 天，
每类最多采集第一页；脚本不会猜测并调用页面中新发现的接口，因为旧式系统
可能用 GET 实现写操作。每组 AJAX 查询会先从自己的模块页提取会话安全令牌，
不会把其他页面的令牌混用于当前接口。
